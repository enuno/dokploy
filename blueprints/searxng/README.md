# SearXNG - Privacy-Respecting Metasearch Engine

[SearXNG](https://github.com/searxng/searxng) is a free internet metasearch engine which aggregates results from more than 70 search services. Users are neither tracked nor profiled.

## Features

- **Privacy-First**: No tracking, no profiling, no cookies required
- **Metasearch**: Aggregates results from 70+ search engines
- **Self-Hosted**: Full control over your search data
- **Customizable**: Enable/disable search engines, set safe search levels
- **No Ads**: Clean search experience without advertisements
- **Open Source**: AGPL-3.0 licensed, community driven
- **Image Proxy**: Results proxied through your instance
- **Fast Caching**: Valkey-backed result caching

## Configuration

### Required Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `SEARXNG_DOMAIN` | Domain for the instance | `search.example.com` |
| `SEARXNG_SECRET` | Secret key for cookies/CSRF | Auto-generated by template |

### Optional Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `SEARXNG_LIMITER` | `false` | Enable rate limiting (recommended for public instances) |
| `SEARXNG_PUBLIC_INSTANCE` | `false` | Enable public instance features |
| `SEARXNG_IMAGE_PROXY` | `true` | Proxy images through SearXNG |
| `SEARXNG_METHOD` | `POST` | HTTP method: `POST` or `GET` |

## Deployment

### 1. DNS Configuration

Create a DNS record:
```
search.example.com → Your Dokploy Server IP
```

### 2. Import Template in Dokploy

1. Navigate to Templates → Import
2. Select the `docker-compose.yml` and `template.toml`
3. Configure `SEARXNG_DOMAIN`
4. Deploy the stack

### 3. Verify Deployment

After deployment:
1. Visit `https://search.example.com`
2. Run a test search
3. Check that results load from multiple engines

## Advanced Configuration

### Customizing Search Engines

To customize which engines are enabled, create a `settings.yml` in the SearXNG config directory (`/etc/searxng/settings.yml`) via a file mount or by accessing the container directly.

See the [SearXNG settings documentation](https://docs.searxng.org/admin/settings/settings_engines.html) for details.

### Enabling the Limiter

For public instances, enable the limiter to reduce bot traffic:

```toml
SEARXNG_LIMITER = "true"
```

### Public Instance Settings

If running a public instance:

```toml
SEARXNG_PUBLIC_INSTANCE = "true"
SEARXNG_LIMITER = "true"
```

## Backup & Restore

### Backup

```bash
# Backup SearXNG cache
docker run --rm -v searxng-data:/data -v $(pwd):/backup alpine \
  tar czf /backup/searxng-backup-$(date +%Y%m%d).tar.gz -C /data .

# Backup Valkey
docker run --rm -v valkey-data:/data -v $(pwd):/backup alpine \
  tar czf /backup/valkey-backup-$(date +%Y%m%d).tar.gz -C /data .
```

### Restore

```bash
# Restore SearXNG cache
docker run --rm -v searxng-data:/data -v $(pwd):/backup alpine \
  sh -c "rm -rf /data/* && tar xzf /backup/searxng-backup-YYYYMMDD.tar.gz -C /data"

# Restore Valkey
docker run --rm -v valkey-data:/data -v $(pwd):/backup alpine \
  sh -c "rm -rf /data/* && tar xzf /backup/valkey-backup-YYYYMMDD.tar.gz -C /data"
```

## Troubleshooting

### Container Won't Start

```bash
# Check logs
docker compose logs searxng

# Verify environment variables
docker inspect <container-id> | grep -A 50 "Env"
```

### Health Check Failing

The health check verifies SearXNG responds on port 8080:
```bash
docker compose exec searxng wget --spider -q http://localhost:8080/
```

### Slow Search Results

1. Verify Valkey is running and connected
2. Check `SEARXNG_VALKEY_URL` is set correctly
3. Review enabled engines — some are slower than others

### No Results from Some Engines

Some search engines block datacenter IPs. If an engine returns no results:
1. Check the engine status in SearXNG preferences
2. Try enabling alternate engines
3. Consider using a VPN or proxy for blocked engines
