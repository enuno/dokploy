# Open Notebook - AI-Powered Notebook for Researchers

[Open Notebook](https://github.com/lfnovo/open-notebook) is an open-source AI-powered notebook designed for researchers. It helps you organize knowledge, synthesize insights from multiple sources, and generate research outputs using modern LLMs.

## Features

- **AI-Powered Research**: Summarize papers, extract insights, and generate content
- **Multi-Source Ingestion**: Import PDFs, web pages, audio, video, and text files
- **Knowledge Graph**: Automatically build connections between sources and notes
- **Collaborative Workspace**: Share notebooks and collaborate with others
- **LLM Integration**: Connect to OpenAI, Anthropic, Google AI, Groq, and Ollama
- **Self-Hosted**: Full control over your research data and AI providers
- **Modern UI**: Clean, responsive interface built with Next.js and Tailwind CSS

## Prerequisites

- Dokploy installed and running
- DNS record pointing to your Dokploy server

## Configuration

### Required Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `OPEN_NOTEBOOK_DOMAIN` | Domain for the instance | `notebook.example.com` |
| `OPEN_NOTEBOOK_ENCRYPTION_KEY` | Secret key for credential encryption | Auto-generated by template |

### Optional Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `CHUNK_SIZE` | `1500` | Text chunk size for embeddings |
| `CHUNK_OVERLAP` | `150` | Overlap between chunks |

### AI Provider API Keys (Configured via UI)

After deployment, configure AI providers in **Settings вҶ’ API Keys**:

- OpenAI
- Anthropic (Claude)
- Google AI (Gemini)
- Groq
- Ollama (self-hosted)

## Deployment

### 1. DNS Configuration

Create a DNS record:
```
notebook.example.com вҶ’ Your Dokploy Server IP
```

### 2. Import Template in Dokploy

1. Navigate to Templates вҶ’ Import
2. Select the `docker-compose.yml` and `template.toml`
3. Configure `OPEN_NOTEBOOK_DOMAIN`
4. Deploy the stack

### 3. First Run Setup

After deployment:
1. Visit `https://notebook.example.com`
2. Add your AI provider API keys in Settings
3. Start creating notebooks and importing sources

## Backup & Restore

### Backup

```bash
# Backup Open Notebook data
docker run --rm -v open-notebook-data:/data -v $(pwd):/backup alpine \
  tar czf /backup/open-notebook-data-$(date +%Y%m%d).tar.gz -C /data .

# Backup SurrealDB
docker run --rm -v surrealdb-data:/data -v $(pwd):/backup alpine \
  tar czf /backup/surrealdb-$(date +%Y%m%d).tar.gz -C /data .
```

### Restore

```bash
# Restore Open Notebook data
docker run --rm -v open-notebook-data:/data -v $(pwd):/backup alpine \
  sh -c "rm -rf /data/* && tar xzf /backup/open-notebook-data-YYYYMMDD.tar.gz -C /data"

# Restore SurrealDB
docker run --rm -v surrealdb-data:/data -v $(pwd):/backup alpine \
  sh -c "rm -rf /data/* && tar xzf /backup/surrealdb-YYYYMMDD.tar.gz -C /data"
```

## Troubleshooting

### Container Won't Start

```bash
# Check logs
docker compose logs open-notebook

# Verify environment variables
docker inspect <container-id> | grep -A 50 "Env"
```

### Health Check Failing

The health check verifies the web UI responds on port 8502:
```bash
docker compose exec open-notebook curl -f http://localhost:8502/
```

If failing, check that the API and SurrealDB are both starting correctly:
```bash
docker compose logs surrealdb
```

### SurrealDB Connection Errors

1. Verify SurrealDB is running: `docker compose ps surrealdb`
2. Check SurrealDB logs: `docker compose logs surrealdb`
3. Ensure both services are on the same network (`open-notebook-net`)

### Cannot Save API Keys

Ensure `OPEN_NOTEBOOK_ENCRYPTION_KEY` is set and at least 16 characters long. If you change this key after deployment, all previously stored credentials will become unreadable.

### Slow Document Processing

1. Check your LLM provider rate limits
2. Reduce `CHUNK_SIZE` for lower memory usage
3. Verify your embedding provider is responding

## Advanced Configuration

### Ollama Integration

To use a local Ollama instance for LLM inference:

1. Deploy Ollama on the same Dokploy server or another accessible host
2. In Open Notebook settings, add your Ollama endpoint (e.g., `http://ollama:11434`)
3. Select an Ollama model for chat or embeddings

### Custom CORS Origins

If you access the UI from multiple domains, set `CORS_ORIGINS` as a comma-separated list:

```toml
CORS_ORIGINS = "https://notebook.example.com,https://www.example.com"
```

### Password Protection

Set an instance-wide password:

```toml
OPEN_NOTEBOOK_PASSWORD = "your-secure-password"
```
